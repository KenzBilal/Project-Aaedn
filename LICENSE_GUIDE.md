# License Guide

Aaedn is not publicly licensed yet. Before publishing, choose a license that matches your distribution intent.

## Quick Recommendation

If you want broad adoption and easy community contribution, use **MIT**.

If you want stronger patent protection and contribution safeguards, use **Apache-2.0**.

## Option Comparison

| License | Good For | Tradeoff |
|---|---|---|
| MIT | Maximum simplicity and adoption | Minimal legal protections |
| Apache-2.0 | Open collaboration with explicit patent grant | Slightly longer legal text/compliance |
| GPL-3.0 | Ensuring derivatives remain open source | More restrictive for some commercial adopters |

## Suggested Path for Aaedn

Given your goals (public use now, potential distribution at production stage):

1. Start with **MIT** or **Apache-2.0** before first public release.
2. Add `LICENSE` file at repository root.
3. Update `README.md` badge and license section.
4. Keep third-party/tooling licenses documented if dependencies are added later.

## Next Step

After you choose a license, replace this guide with:

- a root `LICENSE` file containing full license text
- optional `NOTICE` file if required by license choice
